# 薬物動態カーブビューア

PMDA電子添文XMLから薬物動態パラメータを読み取り、薬の血中濃度が時間とともにどう変わるかを概算表示するWebアプリです。

このアプリは日本のPMDA公開情報を前提にした、日本語・日本国内向けの教育用可視化ツールです。医療判断、服薬指示、用量調整、診断、治療方針の決定には使えません。

## 何をするアプリか

- `https://www.pmda.go.jp` の `ResultDataSetXML` URLを入力として受け取ります。
- PMDA電子添文XMLをNext.js API経由で取得し、ZIP内のXMLを展開して解析します。
- `16. 薬物動態`、特に `16.1 血中濃度` から `Cmax`、`tmax`、`t1/2`、`AUC` などの候補値を抽出します。
- 添文の標準用法を表示し、服用時刻と表示日数を変えた概算曲線を描きます。
- 基本表示は標準用法の曲線だけです。観察用の比較線は、ユーザーが明示的にオンにした時だけ重ねます。
- 使った近似式、抽出元URL、PMDA添文ページへのリンク、語録辞典を画面内に表示します。

## 何をしないか

- 薬を飲むべきかどうかを判断しません。
- 飲む量、回数、間隔、時刻を推奨しません。
- 「この飲み方なら安全」「この血中濃度なら危険ではない」といった判断をしません。
- PMDA XMLを必ず正しく自動抽出できることを保証しません。
- 個人の腎機能、肝機能、年齢、体重、併用薬、遺伝的差異、疾患状態を精密に反映しません。
- 実測血中濃度、医師・薬剤師の判断、添付文書そのものを代替しません。
- 医療機器または臨床意思決定支援ツールとして扱いません。

## 入力URLの探し方

PMDAの添付文書検索で薬を開き、添文ページにあるXMLデータのリンク、または `ResultDataSetXML` を含むURLを入力欄に貼り付けます。

- [PMDAホーム](https://www.pmda.go.jp/)
- [添付文書検索](https://www.pmda.go.jp/PmdaSearch/iyakuSearch/)

例:

- [アレグラ錠30mg／アレグラ錠60mg 添文ページ](https://www.pmda.go.jp/PmdaSearch/iyakuDetail/780069_4490023F1024_1_25)
- [同 XML データセット](https://www.pmda.go.jp/PmdaSearch/iyakuDetail/ResultDataSetXML/780069_4490023F1024_1_25)

アプリ側ではPMDA以外のURLを拒否します。

## XML読み取りの限界

PMDA XMLはPDFやHTMLより構造化されていますが、添文ごとに表の作り、本文の書き方、単位、脚注、行結合・列結合、未変化体と代謝物の扱いが異なります。そのため、XML URLを入力しても必ず `Cmax`、`tmax`、`t1/2` を自動抽出できるとは限りません。

候補が出ない場合や、候補名・単位・数値が不自然に見える場合は、PMDA添文ページやXML本文を人間が確認する必要があります。このアプリの抽出結果は、添文そのものではなく、添文から機械的に読み取った候補値です。

## 現在の主な機能

- PMDA XML URLの入力と解析
- PMDA添文ページ、XMLデータ、PMDA関連リンクの表示
- 標準用法の表示
- 代表的な薬のサンプルカード
- 血中濃度曲線の表示
- `Cmax` の水平線表示
- 標準用法と観察条件の比較表示
- 服用時刻のゆらぎプリセット
- 表示日数の変更
- 使った近似式の表示
- 抽出値、薬効分類、効能・効果、作用機序の抜粋表示
- PMDA公開情報の出典・加工表示
- `PK`、`Cmax`、`tmax`、`ka`、`ke` などの語録辞典

サンプルカードは、その薬を勧めるためのものではありません。薬物動態の形を眺めるための入口です。

## 近似モデル

経口投与後の濃度変化を、一次吸収・一次消失の1コンパートメントモデルで近似します。単回投与の曲線は、立ち上がってから指数関数的に下がる形です。

```text
ke = ln(2) / t1/2

C_single(t) = F * S * (exp(-ke * t) - exp(-ka * t))
```

- `ke`: 消失速度定数。半減期から計算します。
- `ka`: 吸収速度定数。添文の `tmax` に合うよう推定します。
- `F`: 1回量倍率です。
- `S`: 単回投与のピークが添文の `Cmax` に合うようにする補正係数です。

反復服用は、各服用時刻からの単回投与曲線を足し合わせます。

```text
C_total(t) = sum(C_single(t - d_i))
```

この近似は、個人差、活性代謝物、非線形性、徐放剤、貼付剤、複雑な注射剤、食事影響、臓器機能、相互作用などを単純化しています。

## 比較表示について

「同じ1日量なら飲み方を変えても同じなのか」「飲む時間が少しゆらぐと曲線はどう変わるのか」を眺めるため、観察用の比較条件を重ねられます。

ただし、比較表示は服用方法の提案ではありません。同じ1日量でも、回数を減らして1回量を増やすとピーク濃度が高くなり、谷も深くなることがあります。飲み方を変えてよいか、安全かどうかは、このグラフだけでは判断できません。

## 二次利用とデータの扱い

出典は独立行政法人医薬品医療機器総合機構（PMDA）ウェブサイトです。本アプリはPMDA公開情報をもとに薬物動態パラメータを抽出・加工して概算可視化したもので、PMDAまたは製造販売業者による公式な計算結果ではありません。

このリポジトリには、PMDAのPDF/XML本体を保存・同梱しません。アプリは入力されたURLを必要時に取得します。PMDA由来情報の利用にあたっては、PMDAサイトポリシーと公共データ利用規約を確認してください。

- [PMDAサイトポリシー](https://www.pmda.go.jp/0048.html)
- [公共データ利用規約](https://www.digital.go.jp/resources/open_data/public_data_license_v1.0)

## ローカル開発

```bash
npm install
npm run dev
```

ブラウザで `http://localhost:3000` を開きます。

品質確認:

```bash
npm run lint
npm run build
```

## デプロイ

Next.js App Routerの通常構成です。VercelなどのNext.js対応ホスティング環境では、標準的なNode.jsランタイムでビルドできます。

PMDA XML取得はブラウザ直取得ではなく、`/api/parse-pmda` からサーバー側で行います。これによりCORSを避け、サーバーサイドで同じ構成を保てます。

## 主なファイル

- `src/components/pk-viewer.tsx`: 画面UI、サンプル薬、語録辞典
- `src/lib/pmda-parser.ts`: PMDA XML/表/本文の解析
- `src/lib/pk-model.ts`: 薬物動態の近似計算
- `src/app/api/parse-pmda/route.ts`: PMDA XML取得API
- `src/app/globals.css`: 画面スタイル
- `AGENTS.md`: 開発時の方針と医療安全上の実装ルール
