# AGENTS.md

このリポジトリは、日本のPMDA電子添文XMLを読み、薬物動態パラメータから概算の血中濃度曲線を可視化するWebアプリです。ユーザー向けUI、README、コメント、説明文は日本語を基本にします。

このアプリは教育・理解補助のための可視化ツールです。医療判断、服薬指示、用量調整、診断、治療方針の決定に使えるものとして実装・説明してはいけません。

## 作業前の前提

- 主な入力は `https://www.pmda.go.jp` の `ResultDataSetXML` URLです。
- PMDA XMLはZIPとして返ることがあり、中に添文XMLと画像ファイルが含まれます。
- PDF解析は原則として避け、構造化されたXMLを優先します。
- 薬物動態パラメータは添文上の集団値・試験条件値であり、個人予測ではありません。
- グラフは概算シミュレーションです。安全性や有効性を判定するものではありません。
- サンプル薬は推奨ではなく、可視化例として扱います。

この前提に合わない作業が必要になった場合は、実装前に衝突点を明示してください。

## 実装方針

- 変更は依頼範囲に絞ります。
- 未依頼のアカウント機能、保存機能、履歴、DB、分析、通知、外部連携を追加しません。
- 既存のUI文体、色、余白、カード幅、医療安全コピーに合わせます。
- XMLや表は、可能な限りパーサと構造化処理で扱います。
- ただしPMDA本文の定型的な薬物動態文を補助的に読む場合は、対象を狭くして過剰な一般化を避けます。
- 解析、薬物動態モデル、UI表示は混ぜすぎないようにします。
- 1つの薬だけに都合のよい値を汎用ロジックへ直書きしません。
- 単位を明示します。`ng/mL`、`ug/mL`、時間、分、日、投与量を暗黙に混ぜません。
- PMDA以外のURLは取得しません。

## 医療安全

- UIには、医療判断に使えないことを明示します。
- 「この飲み方がよい」「これなら安全」「この濃度なら問題ない」と書きません。
- 比較機能は観察用とし、服用方法の提案に見えないようにします。
- 標準用法以外の条件は、初期表示で自動的に重ねません。ユーザーが明示的にオンにした時だけ比較表示します。
- 治療域や中毒域などを出す場合は、必ず出典を明示し、一般化しません。
- 抽出信頼度が低い場合は、黙ってもっともらしく描くより、候補なし・要確認として扱います。
- 添文の注意、食事影響、腎機能、肝機能、高齢者、小児、相互作用などは、見つかる範囲で限界として扱います。

## PMDA XML解析メモ

優先して見る場所:

- `Pharmacokinetics`
- `BloodLevel`
- `Absorption`
- `Distribution`
- `Metabolism`
- `Excretion`
- `SpecificPopulation`
- `DrugAndDrugInteractions`

表は主に以下の構造で現れます。

- `TblBlock`
- `SimpleTable`
- `SimpTblRow`
- `SimpTblCell`

表を読む時は、行結合・列結合を展開してから列を解釈します。`Cmax`、`tmax`、`t1/2`、`AUC` などの表記ゆれに注意してください。

本文から候補を作る場合は、対象を `16.1 血中濃度` の定型文に限定し、抽出元が本文であることをラベルに残します。

## モデル方針

- `Cmax`、`tmax`、`t1/2` が取れる場合は、一次吸収・一次消失の1コンパートメントモデルを使います。
- `ke = ln(2) / t1/2` として消失速度定数を求めます。
- `ka` は `tmax` に合うよう数値的に推定します。
- 単回投与ピークが添文の `Cmax` に合うようスケールします。
- 反復投与は、各投与時刻からの単回投与曲線を足し合わせます。
- 1回量倍率は線形比例の簡略仮定であり、その旨をUIに出します。
- 徐放剤、貼付剤、複雑な注射剤、活性代謝物、非線形薬物動態は、この近似では単純化されます。

## UI方針

- 最初の画面はランディングページではなく、実際に使うツールにします。
- 主役は血中濃度グラフです。
- 代表的な薬のカードは、薬の推薦に見えない文言にします。
- 内部バージョン名を表に出しません。
- 専門語はなるべく語録辞典や短い補足に逃がし、画面本文を読める状態に保ちます。
- PCとスマホ幅で、文字が枠からはみ出さないように確認します。

## 検証

意味のある変更をしたら、少なくとも以下を確認します。

```bash
npm run lint
npm run build
```

UIに触る変更では、ブラウザで以下を確認します。

- ページが空白でない
- Next.js等のエラーオーバーレイが出ていない
- コンソールに関連エラーがない
- 主要操作が期待通りに状態を変える
- PC幅とスマホ幅で主要UIが崩れない

PMDA解析に触る変更では、少なくとも1つ以上の実URLで候補値が取れることを確認します。
